    1.2 HTML och CSS – bygg vidare på din första sida¶

    I den här modulen bygger vi vidare på den sida du skapade i 1.1 med GitHub Pages.

    Nu går vi från “bara text” till att:

    • förstå grunderna i HTML (struktur och innehåll)
    • se hur CSS styr utseendet
    • göra din sida lite mer lik en riktig personlig/förenings-hemsida

    Smakprov – HTML in 100 seconds (Fireship)¶

    Som en kort, kul introduktion rekommenderar vi videon:

    • HTML in 100 seconds – Fireship

    Mål¶

    Efter den här modulen ska du:

    • känna igen de vanligaste HTML-taggarna:
    • <html>, <head>, <body>
    • rubriker: <h1>–<h3>
    • stycken: <p>
    • länkar: <a href="...">
    • listor: <ul>, <li>
    • veta var du kan skriva CSS för att ändra utseendet
    • kunna:
    • ändra rubriker och text
    • lägga till en länk
    • ändra någon enkel stil (t.ex. färg eller typsnitt)

    Fokus är fortfarande att våga prova, inte att skriva perfekt kod.

    Upplägg på träffen¶

    Föreslaget upplägg för en träff (ca 2 × 45 min med paus):

    Del 1 – Från text till HTML¶

    Kort prat i helgrupp:

    • Vi utgår från sidan du gjorde i 1.1.
    • Kursledaren visar ett enkelt exempel på en HTML-sida, t.ex.:

    ```html

    Min första hemsida

    Min första hemsida

    Hej! Jag heter ... och detta är min första enkla sida.

    ```

    Sedan gör ni tillsammans en av två varianter:

    1. Variant A – skapa index.html
    2. I GitHub: Add file → Create new file
    3. Filnamn: index.html
    4. Klistra in HTML-exemplet ovan.
    5. Spara med Commit changes.

    6. GitHub Pages kommer nu att använda index.html som startsida.

    7. Variant B – kursledaren har redan förberett ett index.html

    8. Deltagarna får en färdig enkel sida att börja ändra i.
    9. Ni fokuserar direkt på att ändra innehåll och utseende.

    Välj den variant som passar gruppens tempo bäst.

    Del 2 – Gemensamma ändringar¶

    Gör några saker tillsammans:

    1. Ändra texten i <h1>
    2. Ändra texten i <p> så den berättar något om dig eller din förening
    3. Lägg till en extra paragraf (<p>) under den första

    Visa sedan hur små CSS-ändringar fungerar:

    • ändra background-color
    • ändra color (textfärg)
    • gör rubriken mittcentrerad:

    css h1 { text-align: center; }

    Varje gång:

    • spara/commit
    • vänta en liten stund
    • ladda om GitHub Pages-adressen och se resultatet

    Övningar för deltagarna¶

    Dessa övningar kan deltagarna göra under passet eller hemma mellan träffarna.

    1. Presentera dig själv eller din förening¶

    Lägg in:

    • en huvudrubrik (<h1>)
    • minst två stycken (<p>) där du:
    • berättar vem du är, eller
    • förklarar vad föreningen gör

    2. Skapa en lista¶

    Skapa en punktlista (<ul> + <li>) med t.ex.:

    • aktiviteter du gillar
    • aktiviteter i föreningen
    • saker du vill lära dig i den här kursen

    Exempel:

    <h2>Aktiviteter jag gillar</h2>
<ul>
  <li>Promenader</li>
  <li>Spela bordtennis</li>
  <li>Koda enkla hemsidor</li>
</ul>

    3. Lägg till en länk¶

    Lägg till en länk (<a>) till:

    • föreningens Facebook-sida
    • en webbplats du gillar
    • en sida med information om något du bryr dig om

    Exempel:

    <p>
  Besök gärna
  <a href="https://www.fvk-media.se" target="_blank">FVK Media</a>
  för mer information.
</p>

    4. Testa mer CSS¶

    Vanliga saker att ändra:

    • bakgrundsfärg på sidan (background-color)
    • typsnitt på hela sidan (font-family)
    • färg på rubriken (color)

    Exempel:

    body {
  font-family: Arial, sans-serif;
  background-color: #202020;
  color: #f5f5f5;
}

h1 {
  color: #a7dca5;
}

    Mellan träffarna (frivillig fördjupning)¶

    För de som vill fortsätta hemma:

    • Titta på några korta videor ur spellistan
      “HTML och CSS – nybörjarguide på svenska”
      https://youtube.com/playlist?list=PLuc3JXzcNTZSYxj8Q32SIbR7wt7fgi7of

    Förslag:

    1. Välj 2–3 videor som verkar lagom.
    2. Pausa ofta och prova att göra samma sak med din egen sida.
    3. Skriv ner:
    4. minst en sak du lärde dig, och
    5. minst en fråga eller något som var oklart.

    Ta med anteckningarna till nästa träff – de kan bli bra utgångspunkt för gemensamt snack.

    Tips till kursledare¶

    • Påminn deltagarna om att inget “går sönder”:
    • blir det konstigt kan ni backa en commit eller byta tillbaka innehållet
    • Visa gärna själv hur du:
    • provar något
    • råkar göra fel
    • rättar felskrivningen
    • Det viktigaste här:
    • att deltagarna känner att de kan öppna, ändra, spara och se resultatet
    • inte att HTML/CSS blir perfekt strukturerat redan nu
