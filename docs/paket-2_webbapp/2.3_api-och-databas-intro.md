# 2.3 API och första steget mot databas

## Mål

Efter den här modulen ska du:

- förstå vad det innebär att **hämta data från ett API**
- känna igen ett enkelt HTTP-anrop i stil med `fetch(...)`
- ha en bild av hur data rör sig: databas → backend → API → frontend
- veta vad en enkel, liten databas kan vara bra till

## Innehåll

### Dataflödet – från databas till skärm

En förenklad kedja kan se ut så här:

1. **Databas**  
   – lagrar aktiviteterna i en tabell.
2. **Backend**  
   – hämtar rader från databasen.
3. **API-endpoint**  
   – t.ex. `/api/aktiviteter` som svarar med JSON.
4. **Frontend**  
   – skickar ett anrop, får JSON tillbaka, visar med komponenter.

### Enkelt API-anrop (pseudokod)

Exempel i JavaScript (princip, inte exakt syntax för något specifikt ramverk):

```js
async function hamtaAktiviteter() {
  const svar = await fetch("/api/aktiviteter");
  const data = await svar.json();
  console.log(data); // en lista av aktiviteter
}
```

Frontend kan sedan:

- spara `data` i en variabel/state
- loopa igenom och visa varje aktivitet i ett kort

### Vad är en enkel databas?

- Tänk tabell i ett kalkylblad:
  - kolumner: `namn`, `plats`, `tid`, `nivå`
  - rader: varje aktivitet
- En databas (t.ex. Postgres, SQLite) lagrar detta på ett säkert och effektivt sätt.
- I början kan man också låtsas att en **JSON-fil** eller en lista i koden är ”mini-databas”.

### Gratisnivåer (överyta)

Det finns tjänster som erbjuder små databaser gratis (”alltid-gratis”-nivåer) för:

- hobbyprojekt
- test och lärande

Den här kursen går inte in på detaljerna, men idén är:

- du kan börja med en väldigt liten databas utan kostnad
- om din app växer → uppgradera eller flytta till egen server

  - Om du hade en tabell med dina aktiviteter – vilka kolumner hade du velat ha?
  - Skriv ner minst 4 kolumner och 3 rader (t.ex. på papper eller i ett kalkylblad).
