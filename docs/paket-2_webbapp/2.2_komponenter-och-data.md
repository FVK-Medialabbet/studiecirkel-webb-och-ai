# 2.2 Komponenter och data i frontend

## Mål

Efter den här modulen ska du:

- förstå idén med **komponenter** i frontend
- kunna föreställa dig din sida uppdelad i mindre delar
- ha sett ett enkelt exempel där data (en lista) visas via en komponent
- känna till hur data kan ligga i en array/objekt i koden

> **Obs:** Vi håller det här på introduktionsnivå.  
> Det viktiga är att förstå *tanken*, inte att kunna all syntax utantill.

## Innehåll

### Vad är en komponent?

- En komponent är en **återanvändbar byggsten** i gränssnittet:
  - meny
  - kort för en aktivitet
  - sidfot
- I ramverk (t.ex. Svelte, React m.fl.) är varje komponent:
  - en fil
  - med HTML-liknande struktur
  - där vi kan stoppa in data

### Dela upp din sida i komponenter

Tänk på din hemsida från paket 1:

- Vad skulle kunna vara egna komponenter?
  - `Header` (logga + meny)
  - `ActivityCard` (ett kort per aktivitet)
  - `Footer` (kontaktinfo)

Bara att *tänka så* är ett steg mot webbapp.

### Data i en lista (array)

Exempel på lista med aktiviteter i pseudokod:

```js
const aktiviteter = [
  {
    namn: "Promenad i Slottsskogen",
    plats: "Slottsskogen",
    tid: "Ons 18:00"
  },
  {
    namn: "Bordtennis",
    plats: "Aktivitetshus Hisingen",
    tid: "Mån 15:00"
  }
];
```

I ett komponent-baserat frontend-ramverk skulle du:

- ha en komponent som tar emot `aktivitet` som ”prop”
- loopen igenom listan och visa ett kort per aktivitet

### Koppling till webbappen

- När data ligger i en lista/array i koden:
  - blir det lättare att byta ut källan (senare: API/databas)
  - du kan sortera, filtrera, begränsa visning m.m.

  - följ en kort introduktionsvideo till Svelte/React där de visar en lista med kort/komponenter.
