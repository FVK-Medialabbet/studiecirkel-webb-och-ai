# 2.1 Från hemsida till webbapp – frontend och backend

## Mål

Efter den här modulen ska du:

- förstå skillnaden mellan **hemsida** och **webbapp**
- ha en enkel bild av vad **frontend** respektive **backend** är
- känna till vad ett **API** gör i mitten
- se hur din egen hemsida skulle kunna bli mer ”app-lik”

## Innehåll

### Hemsida vs webbapp

- **Hemsida:**
  - mest statiskt innehåll
  - HTML + CSS + lite JavaScript
  - uppdateras oftare av redaktör än av användaren
- **Webbapp:**
  - mer interaktiv
  - användaren kan logga in, spara data, göra val
  - mycket av logiken ligger i JavaScript/ramverk och/eller backend-server

### Frontend

- Det användaren ser och klickar på:
  - HTML, CSS, JavaScript
  - ofta med ramverk som t.ex. Svelte, React eller Vue
- Ansvar:
  - visa data på ett tydligt sätt
  - ta emot input (knappar, formulär)
  - prata med backend via API

### Backend

- Kod som körs **på servern**:
  - tar emot anrop från frontend
  - pratar med databasen
  - bestämmer vad användaren får göra
- Kan vara skriven i många språk (t.ex. JavaScript/Node, Python, etc.)

### API – ”översättaren” mellan frontend och backend

- API = ett sätt för program att prata med varandra
- Vanlig form: HTTP-anrop som ger tillbaka JSON-data
- Exempel:
  - frontend frågar: ”ge mig alla aktiviteter”
  - backend svarar med en lista i JSON

### Var passar databas in?

- Backend hämtar/sparar data i en databas
- Exempel: tabell med aktiviteter: `namn`, `plats`, `tid`, `nivå`

  - eller ”what is an API?” på YouTube
