# Övning & självstudier – aktivitetslista som webbapp

## Uppgift

Utgå från idén att bygga en **aktivitetslista som webbapp**:

- en enkel lista över aktiviteter
- varje aktivitet har minst:
  - namn
  - plats
  - dag/tid

Målet är inte att få allt perfekt, utan att:

- tänka i komponenter
- tänka i data
- se framför dig hur API/databas skulle kunna kopplas på

## Steg (konceptuellt)

1. **Bestäm data**
   - skriv en liten lista med 5–10 aktiviteter
   - varje aktivitet = ”rad” med fält, t.ex.:
     - `namn`, `plats`, `dag`, `tid`, `nivå`
2. **Tänk komponenter**
   - hur skulle ett ”aktivitetskort” kunna se ut?
   - vad ska stå på kortet?
3. **Tänk API**
   - föreställ dig en adress, t.ex. `/api/aktiviteter`
   - vad skulle den returnera i JSON?
4. **Koppla ihop**
   - rita ett litet dataflöde:
     - databas → backend → API → frontend → användarens skärm

> I den här modulen räcker det att du **planerar och skissar**.  
> I senare moduler och/eller på egen hand kan du börja implementera det i kod.

## Förslag på vidare självstudier

Om du vill ta det ett steg längre på egen hand:

- Följ en kort tutorial där någon bygger:
  - en ”todo app”
  - eller en enkel ”list-app” med filter
- Försök byta ut deras exempel-data mot din egen aktivitetslista.
- Skriv ner 2–3 frågor du får när du testar – ta med till nästa träff eller dela i Zulip.
