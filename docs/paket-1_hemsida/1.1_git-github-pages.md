## Mål

Efter den här modulen ska du kunna:

- skapa ett konto på GitHub  
- skapa ett nytt repository för din hemsida  
- aktivera GitHub Pages och hitta adressen till din hemsida  
- göra enkla ändringar direkt i webbläsaren och se dem på din sida  

Fokus är att få upp **en första enkel sida på internet**, inte att lära sig all Git-teori ännu.

---

## Varför Git och GitHub (kortfattat)?

Vi går inte på djupet här, men några idéer:

- **Git** är ett system för versionshantering:
  - spara historik  
  - kunna ångra misstag  
- **GitHub** är en plats på internet där:
  - du kan lagra din kod  
  - andra (eller du själv på annan dator) kommer åt projekten  
  - du kan publicera enkla sidor gratis med **GitHub Pages**

Senare i kursen kommer vi tillbaka till Git mer på riktigt.

## Steg 1 – Skapa konto på GitHub

1. Gå till <https://github.com>  
2. Klicka på **Sign up**.  
3. Följ stegen för att skapa ett gratiskonto:
   - e-postadress  
   - lösenord  
   - användarnamn  
   - bekräfta via e-post  

När du är klar och inloggad ser du din **GitHub-startsida**.

---

## Steg 2 – Skapa ett nytt repository (”projekt”)

Vi skapar ett projekt som bara är din första hemsida.

1. Klicka på **+** uppe till höger → **New repository**.  
2. Fyll i:
   - **Repository name:** t.ex. `min-forsta-hemsida`  
   - **Description (valfritt):** t.ex. `Min första testhemsida från studiecirkeln`  
   - Välj **Public**  
   - Kryssa i **Add a README file**  
3. Klicka på **Create repository**.

Nu har du ett nytt projekt på GitHub. README-filen är bara information om projektet – den använder vi inte som webbsida.

---

## Steg 3 – Skapa din första sida (textfil)

GitHub Pages visar en fil som heter `index.md` eller `index.html` som startsida.  
Vi börjar med en **mycket enkel textfil**.

1. I ditt repo, klicka på knappen **Add file** → **Create new file**.  
2. Skriv filnamnet **`index.md`** högst upp.  
3. I den stora rutan skriver du enkel text, t.ex:

   ```markdown
   # Min första hemsida

   Hej!

   Jag heter … och detta är min första lilla testhemsida från studiecirkeln.
   Här kommer jag att öva på att skriva text och senare även HTML.
   ```

   (Du behöver inte förstå `#` ännu – tänk bara att det gör en rubrik.)

4. Scrolla ned och klicka på **Commit changes** (spara).

Nu ligger din textfil `index.md` i repot.

---

## Steg 4 – Slå på GitHub Pages

Nu ska vi säga åt GitHub att visa `index.md` som en hemsida.

1. Gå till fliken **Settings** i ditt repo.  
2. I menyn till vänster, leta upp **Pages**  
   (ibland ligger det under rubriken *Code and automation*).  
3. Under **Source**:
   - välj branch **`main`**  
   - välj **`/(root)`** eller **`/root`** som mapp (om valet finns).  
4. Klicka på **Save**.

Efter några sekunder/minuter dyker det upp en grön ruta med en länk, t.ex:

> Your site is live at `https://ditt-namn.github.io/min-forsta-hemsida/`

Klicka på länken – nu ser du din **första hemsida**!

---

## Steg 5 – Uppdatera sidan (din nya ”spara-knapp”)

Varje gång du vill ändra texten gör du så här:

1. Gå tillbaka till ditt repo → klicka på filen **`index.md`**.  
2. Klicka på **pennan** (Edit this file).  
3. Ändra texten, t.ex. lägg till en ny rad om dig själv.  
4. Scrolla ned och klicka på **Commit changes**.

Vänta någon halv minut och **ladda om** din webbadress – sidan uppdateras.

> Tänk på “Commit changes” som en lite mer högtidlig **spara-knapp**.  
> Vi kommer senare prata om vad en *commit* egentligen är och vad *Git* gör i bakgrunden.

---

## Vad händer i nästa sittning?

Nästa gång börjar vi:

- titta på **HTML-taggar** (rubriker, stycken, länkar)  
- stegvis ersätta din enkla text med **riktig HTML**  
- förstå lite mer hur webbläsaren ”läser” koden  

Du kan redan nu leka lite genom att:

- ändra texten i `index.md`  
- skriva någon mening om:
  - vad du vill lära dig i kursen  
  - vilken typ av webb eller app du drömmer om  

Spara med **Commit changes** och se hur din sida ändras.

---

## Förslag på självstudier

- Gör minst **två små ändringar** i `index.md` mellan träffarna, t.ex.:
  - lägg till mer text  
  - ändra rubriken  
- Titta gärna på **commit-historiken** i ditt repo:
  - klicka på fliken **Commits**  
  - se hur GitHub sparar varje ändring som ett steg i historiken
