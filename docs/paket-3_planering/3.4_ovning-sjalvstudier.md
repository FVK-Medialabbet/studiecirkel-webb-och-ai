## Uppgift

Målet med den här övningen är att skapa en **enkel projektplan på 1–2 sidor** för en idé du bryr dig om.

Du kan använda valfri form (markdown-fil, Google Docs, papper), men gärna:

- en `PROJECT_PLAN.md` i ditt repo  
  *eller*  
- en sida i `docs/`-mappen

## Innehåll som ska finnas med

1. **Idé och bakgrund**
   - kort beskrivning (2–4 meningar)
   - målgrupp
   - vilket problem du vill lösa

2. **Målbild**
   - hur ser det ut när första versionen (v0.1) är ”tillräckligt bra”?
   - gärna i punktform

3. **User stories**
   - minst 3, gärna 5–7
   - skrivna på enkelt språk (”Som X vill jag Y så att Z”)

4. **Prioritering / scope**
   - markera vilka stories som ingår i:
     - v0.1 (nu)
     - v0.2 (snart)
     - v0.3 (senare)

5. **Repo och arbetssätt**
   - länken till ditt repo (om det finns)
   - kort text om hur du tänker jobba:
     - ensam eller i liten grupp?
     - hur du vill använda issues/branches/pull requests?

## Förslag på vidare självstudier

- Skapa ett nytt repo på GitHub för projektet om du inte redan gjort det.
- Lägg in projektplanen som `PROJECT_PLAN.md` eller i `docs/`.
- Skapa issues för de viktigaste user stories du vill börja med.
- Dela repo-länken och ev. frågor i Zulip-kanalen så andra kan ge feedback.
