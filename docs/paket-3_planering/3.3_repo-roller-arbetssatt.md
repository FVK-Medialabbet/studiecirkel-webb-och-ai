# 3.3 Repo, roller och arbetssätt i ett litet team

## Mål

Efter den här modulen ska du:

- förstå hur din projektplan kan översättas till ett **repository** på GitHub
- ha en idé om vilka **roller** som behövs (även om ni är få personer)
- känna till ett enkelt sätt att jobba med issues, branches och pull requests

## Innehåll

### Repo som projektets nav

Ett repository (repo) på GitHub kan samla:

- kod
- dokumentation (t.ex. din projektplan i `docs/` eller `README.md`)
- issues (uppgifter, förbättringar, buggar)
- diskussioner och historik

Minsta innehåll:

- `README.md` med:
  - kort beskrivning
  - hur man startar/testar projektet
  - målgrupp/målbild i några meningar
- en mapp för dokumentation (`docs/` eller liknande)

### Roller i ett litet team

Även om ni bara är 1–3 personer kan ni tänka i roller:

- **Produkt-/idéansvarig** – håller koll på målbild, prioriteringar
- **Utvecklare** – skriver kod, testar, gör ändringar
- **Dokumentatör/testare** – ser till att README och instruktioner hänger med, provar som ”användare”

I praktiken kan samma person ha flera roller, men det blir tydligare när ni pratar om projektet.

### Enkelt arbetssätt med GitHub

Förslag på enkel process:

1. Skapa en **issue** för en uppgift (t.ex. en user story eller del av den).
2. Skapa en **branch** med ett kort namn som beskriver uppgiften.
3. Gör ändringar i branchen och pusha.
4. Öppna en **pull request** när du är nöjd.
5. Låt någon (eller du själv vid senare tillfälle) läsa igenom och godkänna.
6. Mergar in i `main` när det ser bra ut.

Det här ger:

- historik kring varför saker ändrats
- ett naturligt ställe att kommentera och diskutera lösningar

## Förslag på självstudier

- Skapa ett nytt repo (eller använd ett befintligt) och:
  - skriv en enkel `README.md` som beskriver projektet
  - skapa minst 3 issues baserade på dina user stories
- Om ni är flera:
  - bestäm vem som tar vilken roll för nästa lilla del (t.ex. v0.1)
  - prova att jobba via issue + branch + pull request även om det känns ”överdrivet litet” – det är träning.
